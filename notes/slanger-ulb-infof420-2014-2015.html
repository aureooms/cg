Lecture 1: 3/11/2014

  • Course description.
  • What is Computational Geometry?
  • Models of computation:
    • RAM model (Lecture notes at Yale by Laszlo Lovasz)
    • Real-RAM model
    • Ruler&Compass model (applet by François Labelle). Play the Euclid game.
  • The Dot Product, Cross Product and applications.
  • Right-turn - Left-turn orientation determinant (Notes from Rodrigo Silveira at UPC)
  • Dealing with degeneracies: Simulation of Simplicity
  • Convexity, convex combination: definition.
  • Convex Hull $CH(S)$ of a finite set $S$ of points.
    • Definition: $CH(S)$ is the intersection of all convex sets containing $S$.
    • Equivalent definition: The union of all convex combinations of points of $S$.
    • Equivalent definition: The union of all convex combinations of triples of points (triangles) of $S$ using Carathéodory's Theorem.
    • definition of a vertex of $CH(S)$ (a "corner" of the boundary).
    • A pt is a vertex iff it is not inside any triangle => $O(n^4)$ algo.
    • The order of the vertices along $CH(S)$ can be found by sorting the vertices using the orientation determinant as a comparison operator.
  • Project: implement (in processing-js, processing or other)

    • test if a point is inside a triangle given by 3 points
    • compute the vertices of $CH(S)$ for a set $S$ of points
    • find the order of the vertices using a sorting algorithm

Lecture 2: 6/11/2014

    • A pt is a vertex iff it is not inside any triangle => $O(n^4)$ algo.

Lecture 4: 13/11/2014, Triangulations, Sweep Line Algorithms

  • Flashback: deciding if a point is inside a (non-convex) polygon.
  • Batching: Deciding which among $n$ points are inside a polygon. Sweepline algorithms.
  • Triangulating a polygon
    • Two ears theorem, method for finding two ears.
    • Proof that triangulation is always possible (ear-cutting and gluing back).
    • Algorithms:
      • $O(n^3)$ triangulation, brute force.
      • $O(n^2)$ triangulation by finding chord in $O(n)$
      • finding ear in $O(n)$.
  • dual tree of a triangulation and 2 leaves = ears.
  • Triangulation of a monotone chain in $O(n)$ time (Graham scan).
  • Monotone polygons, $O(n)$ time Triangulation.
  • Sweepline algorithms. Example: Given $n$ points and and an orthogonal polygon with $n$ vertices, determine which points are in the polygon in $O(n \log n)$ time
  • Example 2: same for a non-orthogonal simple polygon.
  • Review: Balanced trees AVL-Trees by Erik Demaine at MIT/OCW.
  • Next time: Sweepline algorithm to decompose a polygon into monotone pieces.
  • Read: Textbook chapter 3
  • Project:
    • Write a program which, given a polygon finds an ear in $O(n)$ time.,
    • and then triangulate the polygon recursively.
    • Does Graham scan always triangulate the region between a polygon and its convex hull? Prove or give a counterexample.

    • Lecture 8: 1/12/2014. Duality Transforms

    • Duality transforms:
      • Motivation: intersection of halfplanes
      • Point to line duality: $(a,b)$ to $y=ax+b$
      • From the incidence property, derived line to point: $y=cx+d$ to $(-c,d)$
      • Preserves:
        • Incidences
        • Above/below
        • vertical distance
      • Dual vision of
        • Vertical lines to points at infinity (homogeneous coordinates)
        • Collinear points
        • Halfplanes
        • Intersection of halfplanes to convex hull
        • Testing if a polygon is star-shaped = testing if intersection of wedges is non-empty = testing if intersection of halfplanes is empty = 2 convex hull in the dual + testing if they intersect
    • Arrangements:
      • Definitions: vertices, edges, faces
      • Complexity: number of vertices, edges, faces
      • Data structure: DCEL
      • Construction: linesweep -> $O(n^2 \log n)$
      • Construction: incremental
      • Zone theorem: complexity of a zone is $O(n)$ => complexity incremental construction is $O(n^2)$
      • Read: Textbook chapter 8.

    Lecture 9: 4/12/2014. More on Duality. Linear Programming

    • Ham-Sandwich cut theorem. Proof using duality and median levels. Algorithm in $O(n^2)$ using the construction of the arrangement. Mention of faster algorithms.
    • More on duality: polar duality, sphere view of the projective plane and projective duality.
      • Brief mention of higher dimension duality transforms, point to planes and Plucker coordinates.
      • A segment = a double wedge in the dual
      • The convex billiard challenge: hit all edges of a convex polygonal billiard in order with one ball = hit all edges of a polygonal path with a line = finding a common intersection between double wedges in the dual
    • Linear Programming:
      • Definition, Simplex method (can be slow)
      • Applications: recognizing star-shaped polygons, removing a cake from its mold.
      • Linear programming in $\mathbb{R}^1$ in $O(n)$
      • In $\mathbb{R}^2$: Convex Hull -> $O(n \log n)$
      • In $\mathbb{R}^2$: incremental -> $O(n^2)$
      • In $\mathbb{R}^2$: randomized incremental -> $O(n)$ expected
      • In $\mathbb{R}^d$: randomized incremental -> $O(d! n)$ expected
      • Smallest enclosing circle and LP-type problems
      • Read: Textbook chapter 4
    • Project: implement some (or several) dualities showing the correspondance between objects (points, lines, other).
